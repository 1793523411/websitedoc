## 描述

为什么会有这个功能？是这样的，在我以前写代码时难免会使用到图片，可我不太想将图片放到本地项目，然后引用相对目录，一是因为图片本来相比代码就很大，而是觉得这样很别扭，所以我几乎每次引用图片都是用的一个网上链接

web 端有 https 有开头的网址可用，要么随便找个网页在找一张图片右键复制图片链接，或者写一篇博客里面放各种自己喜欢图片，每次想用就打开这篇博客复制图片链接，[我以前在 CSDN 写过的图片博客 😂](https://blog.csdn.net/littleCucumber/article/details/104226295)

微信小程序也可像上面那样，但更方便的是微信小程序自带的云开发

但上面的的那些方式都不是太统一，所有就有了这个功能来统一处理图片：使用对象存储来统一管理图片

有时我会直接往对象存储上直接存图片，然后使用对应的链接，但我遇到的问题是每次都要登录对应的平台，现在的登录还很'慢',扫码登录，然后我每回都得拿出手机，就觉得很麻烦，所以...

使用对象存储的 SDK 和密钥的形式来管理对象存储的图片，当然不仅仅是图片了
